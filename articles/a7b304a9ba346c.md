# それぞれを外部から開かせよう
これから各エディタをそれぞれ連携させていくわけですが、まずは既に開いてる各エディタに対して、CLI から命令を投げる事で指定したファイルを開かせる事ができるかを確認していきます。
これができないと後々連携させるのが面倒なので、確認も兼ねてやっておきましょう。

## CLI -> Neovim
どうやら Neovim は起動時にサバを立てているようで、このサバとやりとりする事で外部から開かせる事ができるみたいです。
このサバは [`v:servername`](https://vim-jp.org/vimdoc-ja/eval.html#v:servername) からアクセスできるようですね。
んで、このサバに命令を投げる方法ですが、`msgpack-rpc` を使って細やかな制御をする方法等があるようです。が、面倒なので、今回は CLI の一行でナンと化する方向でいきます。

具体的には `nvim --server hoge --remote-send "piyo"` みたいにすることで、`hoge` で立ってる nvim サバに対し、`piyo` というキー入力を投げられる方法を使用します。
各サバはデフォルトで `nvim.` のプレフィクスのようなので、該当しそうなやつを一個探してファイルを開くキー入力を投げます。

```powershell:open-in-neovim.ps1
param (
	[string]$FilePath,
	[int]$Line = 1
)

$nvimPipe = Get-ChildItem \\.\pipe\ | Where-Object { $_.Name -match 'nvim\.' } | Select-Object -First 1 -ExpandProperty Name

if (-not $nvimPipe) {
	Write-Error "No running Neovim server found."
	exit 1
}

$server = "\\.\pipe\$nvimPipe"
$command = "<C-\><C-N>:edit +$Line $FilePath<CR>"

nvim --server $server --remote-send $command
```

こんな感じの powershell のスクリプトを書くと、以下のように呼び出すだけで実現ができます。

```
> .\open-in-nvim.ps1 E:\workspace\hoge.txt 5

> .\open-in-nvim.ps1 -FilePath E:\workspace\hoge.txt -Line 5
```

今回は簡単のため、どの nvim で開くかは特に指定していませんが、nvim 起動時に `v:servername` を固定したり、RPC を使ってそれ専用のプログラムを作成したりして、開く nvim を指定するのも良さがありますね。

## CLI -> Visual Studio

Visual Studio は devenv ってやつを使う事で外部から制御できるらしいので、そいつを使って確認してみます。

```
> devenv.exe /Edit E:\workspace\hoge.txt /Command "Edit.Goto 5"
```

どの Visual Studio で開くかの詳細の指定はできませんが、.sln の指定ができますし、ある程度の調整はできそうです。
なによりそれ用のコマンド(devenv) があるのが便利で良いすねぇ。さすが IDE といった貫禄を感じます。

## CLI -> Visual Studio Code

VSCode はよりいっそう簡単に実現できます。
簡単過ぎて書く事が無いくらいです。

```
> code --goto E:\workspace\hoge.txt:5
```

# それぞれから開こう

ここまでで、CLI から各エディタに対して指定のファイルを開かせられる事が分かりました。
ここからは、各エディタから先ほどのコマンドを実行して連携させていきましょう。

## Neovim -> Visual Studio ・ Visual Studio Code

Neovim で実現する方法は色々あるとは思いますが、今回は簡単のため、コマンドを追加する形式にしときます。

### Visual Studio

```vim:open-in-vs.vim
function s:openInVs(file, line) abort
	let devenv = "C:\\Program Files\\Microsoft Visual Studio\\18\\Community\\Common7\\IDE\\devenv.exe"
	if !filereadable(a:file)
		return
	endif
	call jobstart([devenv, "/Edit", a:file, "/Command", "Edit.Goto " .. a:line])
endfunction

command! -nargs=? -complete=file OpenInVs
			\	call s:openInVs(
			\		empty(<q-args>) ? expand('%:p') : <q-args>,
			\		empty(<q-args>) ? line('.') : 1
			\	)
```

### Visual Studio Code

```vim:open-in-vscode.vim
function s:openInVsCode(file, line) abort
	let code = "C:\\Users\\unko\\AppData\\Local\\Programs\\Microsoft VS Code\\Code.exe"
	if !filereadable(a:file)
		return
	endif
	call jobstart([code, "--goto", a:file .. ":" .. a:line])
endfunction

command! -nargs=? -complete=file OpenInVsCode
			\	call s:openInVsCode(
			\		empty(<q-args>) ? expand('%:p') : <q-args>,
			\		empty(<q-args>) ? line('.') : 1
			\	)
```

## Visual Studio -> Neovim ・ Visual Studio Code

Visual Studio では「ツール」>「外部ツール」を設定する事で以下の画像のように発火させられるようです。

![](https://static.zenn.studio/user-upload/3d795d432a2c-20260524.png =250x)

一度ツールからアクセスできるようになると、キーバインドの設定もできるので、まずは「外部ツール」で設定していきましょう。

### Neovim
当初は、pwsh を指定して先ほどの .ps1 をそのまま実行するようにしていたのですが、
それだと謎にコンソール画面が表示されてしまっておりました。

どうやら、VBScript ってやつをかませるとコンソール画面が表示されなくなるようなので、そやつ経由で呼び出されるように一工夫加えてやります。

```vb:open-in-neovim.vbs
Set shell = CreateObject("WScript.Shell")

cmd = "pwsh -NoProfile -ExecutionPolicy Bypass -File ""C:\Users\unko\open-in-nvim.ps1"""

For Each arg In WScript.Arguments
    cmd = cmd & " """ & arg & """"
Next

shell.Run cmd, 0, False
```

| 項目 | 値 |
|---|---|
|タイトル |  `open in neovim` 等のお好きなものをどうぞ |
|コマンド |  `C:\Windows\System32\wscript.exe` |
|引数 |    `"C:\Users\unko\open-in-nvim.vbs" "$(ItemPath)" "$(CurLine)"` |


### Visual Studio Code

こっちはそのままコマンド叩くだけなので、特筆すべき事もないですね。
楽チンx2

| 項目 | 値 |
|---|---|
| タイトル |   `open in vscode` 等のお好きなものをどうぞ |
| コマンド |   `C:\Users\unko\AppData\Local\Programs\Microsoft VS Code\Code.exe` |
| 引数 |   `--goto $(ItemPath):$(CurLine)` |



## Visual Studio Code -> Neovim ・ Visual Studio

VS Code では実行するコマンドをタスクとして登録しておき、それをコマンドパレットから実行するフローにしてみましょう。
一度タスクに登録してしまえば、キーバインドの設定もできるようなので、タスクの登録のみを記載しまう。

まずはコマンドパレットで「Open User Tasks」なりするなどして、適切な tasks.json の tasks に以下をそれぞれ追加しましょう。

### Neovim

タスク経由だと謎ウィンドウが表示されないっぽいので、 ps1 を直接実行してる

```json:tasks.json
{
    "label": "Open in Neovim",
    "type": "shell",
    "command": "pwsh",
    "args": [
        "-NoProfile",
        "-ExecutionPolicy",
        "Bypass",
        "-File",
        "C:\\Users\\unko\\open-in-nvim.ps1",
        "-FilePath",
        "${file}",
        "-Line",
        "${lineNumber}"
    ],
    "problemMatcher": []
}
```

### Visual Studio
特に無し

```json:tasks.json
{
    "label": "Open in Visual Studio",
    "type": "shell",
    "command": "C:\\Program Files\\Microsoft Visual Studio\\18\\Community\\Common7\\IDE\\devenv.exe",
    "args": [
        "/Edit",
        "${file}",
        "/Command",
        "Edit.Goto ${lineNumber}"
    ],
    "problemMatcher": []
}
```
